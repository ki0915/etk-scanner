# $search — 취약점 대상 탐색 프롬프트

## 목적
pypi/npm 유명 패키지 중 분석 가치가 높은 대상을 선정한다.

## 선정 기준 (우선순위 순)
1. 주간 다운로드 수 100만 이상
2. GitHub Stars 높은 순
3. C/C++ 래핑된 Python 패키지 (메모리 취약점 가능성)
4. AI/ML 카테고리 패키지
5. 최근 업데이트가 활발한 패키지 (새 코드 = 새 버그)

## 출력 형식

```
패키지명: <name>
생태계: pypi / npm
GitHub URL: <url>
주간 다운로드: <수>
GitHub Stars: <수>
분석 우선순위 사유: <한 줄>
```

## 제외 기준
- 이미 candidates/ 폴더에 존재하는 패키지 (tracker.py로 확인)
- 기존 CVE가 최근 30일 내 발급된 패키지
- 활발히 유지보수되지 않는 패키지 (마지막 커밋 2년 이상)
